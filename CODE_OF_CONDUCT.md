# Code of Conduct / 行為準則

> A small project, kept friendly. / 小專案，保持友善。

## English

### Spirit

This is a small Traditional Chinese reader-firmware project. We're building tools we ourselves use. Newcomers are welcome; rudeness is not.

### What we expect

- **Be specific.** Vague complaints don't help us help you. "It's broken" is less useful than "Tapping chapter TOC on a 2000-chapter web novel freezes the reader for 30s, here's the serial log."
- **Be patient.** This is a hobby project. Expect days-to-weeks response time.
- **Be a good faith reader.** Read the README, CHANGELOG, and existing issues before opening a new one. Search before reposting.
- **Be honest about scope.** Don't ask for features that require paid services or compromise the core use case.
- **Be kind to upstream.** We owe ChineseType, CrossInk-Carousel, and crosspoint-reader for the foundation. Constructive critique of our changes is fine; trash-talk of upstreams is not.

### What we won't tolerate

- Personal attacks, harassment, or threats
- Doxxing or sharing others' private information
- Spam, off-topic promotion, or repeated rule-breaking
- Bad-faith behavior aimed at extracting work from maintainers

### If something goes wrong

Open an issue tagged `meta` or contact the maintainer directly. We'll listen and act with proportion. Bans are last-resort but available.

### Scope

This applies to all project spaces: GitHub issues, PRs, discussions, and any communication clearly tied to this project.

## 繁體中文

### 精神

小型的繁中 reader 韌體專案。我們做自己用的工具。歡迎新人，但不歡迎粗魯。

### 我們期待

- **講具體。** 模糊抱怨幫不到忙。「壞了」沒「點 2000 章網文的目錄會讓 reader 凍結 30 秒，這是 serial log」實用。
- **有耐心。** 這是業餘專案，回應時間從幾天到幾週。
- **善意讀者。** 開 issue 前讀 README、CHANGELOG、已有的 issue。搜尋過再重發。
- **對 scope 誠實。** 不要要求需要付費服務或傷害核心使用體驗的功能。
- **對上游友善。** 我們欠 ChineseType、CrossInk-Carousel、crosspoint-reader 一個基底。對我們的改動建設性批評可以，但不要罵上游。

### 我們不容忍

- 人身攻擊、騷擾、威脅
- 公開他人隱私 / doxxing
- 灌水、不相關的宣傳、重複違規
- 惡意榨取維護者勞動的行為

### 如果有問題

開一個標 `meta` 的 issue，或直接聯絡維護者。我們會聽、會做合比例的處理。封禁是最後手段但可以動用。

### 範圍

涵蓋所有專案空間：GitHub issues、PR、discussion，以及任何明顯跟本專案有關的溝通。
